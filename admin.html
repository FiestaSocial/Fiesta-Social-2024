    Pasos para verificar la conexión

    ​​​​ < hola >
  • Confirma que el código publicado incluye doPost(e) ( y opcionalmente doGet ( e ) ) como en el ejemplo y que ambos devuelven la respuesta con encabezados CORS .
    • Desde Apps Script, abre Ver ▸ Registros o usa el Registro de ejecución para comprobar si llegan solicitudes después de cada prueba. Los registros deberían mostrar el JSON recibido .
  • Realice una reserva de prueba en el sitio web público; luego actualice la hoja "Datos del huésped" y verifique que aparezca una fila con la fecha del servidor más reciente.
  • Si la cola no se agrega, revise la pestaña Red de las herramientas para desarrolladores del navegador y observe el estado HTTP de la solicitud. Compárelo con los registros de scripts para detectar bloqueos de permisos o errores de análisis .
  • Abre el enlace CSV publicado en otra pestaña y confirma que muestra los datos. Si ves un mensaje de permisos o un error 404/403, vuelve a publicar la página o ajusta el enlace en la configuración .
